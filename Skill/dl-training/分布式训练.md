# 分布式训练

## 概念

分布式训练将模型或数据分配到多个设备上并行计算，是训练大模型的必备技术。

## 主要范式

### 数据并行 (Data Parallelism, DP)

```
复制模型到各设备，分割数据
```

**流程**:
1. 模型复制到各GPU
2. 数据分割为mini-batch
3. 各GPU独立前向+反向
4. All-Reduce同步梯度
5. 各GPU更新参数

**优点**: 实现简单，适合大多数场景
**缺点**: 模型必须能放入单GPU显存

### 模型并行 (Model Parallelism, MP)

```
分割模型到各设备，数据复制
```

- **流水线并行**: 按层分割
- **张量并行**: 按张量维度分割

**优点**: 可训练超大模型
**缺点**: 通信开销大，利用率可能低

### 混合并行

```
数据并行 + 流水线并行 + 张量并行
```

- **应用**: 大规模模型训练（如GPT-3）
- **示例**: 8路数据并行 × 4路流水线 × 2路张量

## 关键技术

### All-Reduce

```
各设备梯度求和并广播
```

- 环形All-Reduce: 带宽高效
- 树形All-Reduce: 延迟低

### ZeRO (Zero Redundancy Optimizer)

**问题**: 数据并行每设备存完整模型、梯度、优化器状态

**ZeRO优化**:
- **Stage 1**: 分片优化器状态
- **Stage 2**: 分片梯度
- **Stage 3**: 分片模型参数

**效果**: 显存占用大幅降低，可训练更大模型

### 流水线并行优化

**问题**: 设备空闲等待（气泡）

**解决方案**:
- **GPipe**: 分割mini-batch为micro-batch
- **1F1B (One Forward One Backward)**: 交替执行减少气泡

### 梯度累积

```
累积多个小batch的梯度后更新
等效大批量训练
```

## 通信后端

| 后端 | 特点 | 适用场景 |
|-----|------|---------|
| NCCL | NVIDIA GPU优化 | 多GPU训练 |
| Gloo | CPU和GPU | 跨平台 |
| MPI | 高性能计算 | 集群环境 |

## 框架支持

### PyTorch DDP

```python
model = DDP(model, device_ids=[local_rank])
# 自动梯度同步
```

### DeepSpeed

- ZeRO优化
- 混合精度训练
- 梯度检查点
- 流水线并行

### Megatron-LM

- 张量并行
- 流水线并行
- 大模型训练框架

### FSDP (Fully Sharded Data Parallel)

- PyTorch原生ZeRO-3实现
- 简化大模型训练

## 实践建议

### 选择策略

| 模型大小 | 显存需求 | 推荐方案 |
|---------|---------|---------|
| 单卡能放下 | 小 | 单机训练 |
| 单卡放不下，多卡能放 | 中 | DDP + 梯度累积 |
| 需要多机多卡 | 大 | DeepSpeed / FSDP |
| 超大模型 | 很大 | 3D并行 |

### 常见问题

- **通信瓶颈**: 使用高速互联（NVLink、InfiniBand）
- **负载不均**: 合理分配数据和模型
- **同步卡住**: 检查死锁和超时
