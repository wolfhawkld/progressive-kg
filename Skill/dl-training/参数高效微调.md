# 参数高效微调（PEFT）

> 冻结大部分预训练参数，只训练适配器、低秩更新或软提示，以降低可训练状态和多任务存储成本。

## 核心边界

PEFT 减少的是可训练参数、对应梯度/优化器状态和每任务增量权重。完整基座通常仍需前向传播，激活显存也不会按可训练参数比例同步下降；吞吐、峰值显存和质量必须分别测量。

它适合低成本适配和多任务部署，但并不保证只训练 0.1% 参数就能达到全量微调质量。

主要方法

各方法分别把可训练容量放在低秩权重、层内瓶颈或连续提示中，因此内存与部署行为不同。

### LoRA

冻结 $W\in\mathbb R^{d_{out}\times d_{in}}$，学习低秩增量：

$$
W'=W+\frac\alpha r BA,
\quad
B\in\mathbb R^{d_{out}\times r},
A\in\mathbb R^{r\times d_{in}}
$$

rank、scale、初始化和 target modules 决定容量。单个 LoRA 可在部署前合并到基座权重以消除额外线性分支；动态切换多 adapter、量化基座或某些 serving 布局下不一定能无代价合并。

### Bottleneck Adapter

在层内插入降维—非线性—升维模块。它保留每任务独立参数，但推理时增加额外层和访存。

### Prefix / Prompt Tuning

- Prefix tuning 为多层注意力学习 prefix K/V 或其参数化表示。
- Prompt tuning 主要在输入 embedding 层学习连续软 token。

两者不是简单的“在文本前加可读 token”，容量和位置也不同。

### QLoRA

QLoRA 以低比特量化形式存储冻结基座，并通过 LoRA 训练增量。它降低基座显存，但计算会涉及反量化 kernel，质量和速度依量化格式与实现。

## 选择步骤

1. 明确任务差异是输出格式、领域知识还是深层行为改变。
2. 建立 full fine-tuning 或冻结线性头等可比基线。
3. 选择 target modules，并从较小 rank 开始做容量曲线。
4. 分别记录 trainable params、峰值显存、step time、checkpoint 大小和验证质量。
5. 在部署路径测试 adapter 合并、并发切换和量化兼容。

| 失败现象 | 可能调整 |
|---|---|
| 明显欠拟合 | 增加 rank/adapter 宽度或扩大 target modules |
| 训练好但迁移差 | 数据配比、正则化、学习率或基座能力不足 |
| 显存仍很高 | 激活检查点、序列长度、batch、量化基座，而非只看参数数目 |
| 多 adapter 服务慢 | 合并策略、batch 分组、kernel 与缓存布局 |

## 关系网络

- 应用 [[Transformer架构]] — 常在注意力投影和 FFN 上插入增量参数
- 组合 [[模型量化]] — QLoRA 将量化基座与 LoRA 结合
- 对比 [[知识蒸馏]] — PEFT 适配同一基座，蒸馏把行为迁移到学生模型
- 依赖 [[训练调试]] — 冻结范围、target modules 和 trainable params 需显式核对
- 权衡 [[帕累托(Pareto)]] — 质量、训练显存、checkpoint 大小和 serving 开销需联合选择
