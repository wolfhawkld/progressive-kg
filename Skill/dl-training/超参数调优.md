# 超参数调优

> 在固定数据协议和计算预算下搜索配置，并用独立验证与重复实验选择可复现的折衷方案。

## 先定义问题

调优不是抽象地寻找“全局最优参数”，而是在有限、带噪预算下选择配置。开始前固定：

- train/validation/test 划分与预处理；
- 主指标、约束和 tie-break；
- 每个 trial 的训练预算与提前停止规则；
- 并行资源和总计算预算；
- 最终复验的随机种子数量。

测试集必须留到方案冻结后使用；反复根据 test 调参会把测试集变成验证集。

## 构造搜索空间

| 参数 | 常见空间 |
|---|---|
| 学习率、weight decay、epsilon | 对数尺度 |
| dropout、mix 概率 | 有界连续尺度 |
| 层数、rank、batch | 离散或条件空间 |
| optimizer/schedule | 类别选择，并带各自条件参数 |

范围应来自短程稳定性测试和资源约束，不应套用跨模型的 `1e-5–1e-1` 大表。条件参数要避免无意义组合，例如只在 cosine schedule 下搜索 cosine 的末值。

## 搜索方法

搜索方法主要在覆盖范围、先验利用和提前淘汰之间权衡，应按维度、噪声和单次训练成本选择。

### Grid 与 Random Search

网格适合低维、明确离散候选；当只有少数维度真正重要时，随机搜索通常比规则网格覆盖更多有效取值。随机搜索的优势不是对任何问题都成立，分布选择同样关键。

### Bayesian Optimization

用代理模型和 acquisition function 选择下一试验，适合单次训练昂贵且维度适中的场景。高维、强条件和高度噪声空间会降低代理质量。

### Successive Halving / Hyperband

先给许多配置小预算，再把资源分配给有希望者。它假设早期表现对最终结果有一定预测力；存在晚起效 schedule 或尺度规律时，过早淘汰会有偏差。

## 推荐流程

1. 先用 [[训练调试]] 确认单个基线正确。
2. 调“科学参数”（模型容量、数据与正则化目标）前，先找到稳定的优化区域。
3. 先做宽范围随机/多保真搜索，再在高概率区域缩小范围。
4. 保留所有 trial 的完整配置、学习曲线、失败原因和资源用量。
5. 对候选配置用多个 seed 和完整预算复验。
6. 冻结方案后只做一次独立 test 评估，并报告选择过程。

## 多目标与公平比较

若同时关心质量、延迟、显存和能耗，应保留 Pareto front，而不是先把所有指标随意加权成一个分数。硬件指标必须在相同设备、batch、输入长度、预热和测量窗口下比较。

失败 trial 也是信息，但要区分数值发散、OOM、超时和基础设施故障；把它们全记成最差质量会扭曲搜索模型。

## 关系网络

- 调整 [[学习率调度]] — 峰值、warmup 和曲线形状是联合空间
- 调整 [[正则化]] — weight decay、dropout 和增强强度相互作用
- 依赖 [[训练调试]] — 实现错误会让搜索优化错误目标
- 使用 [[帕累托(Pareto)]] — 保留质量与资源的非支配方案
- 评估 [[分布式训练]] — 并行策略会改变可行 batch 与吞吐，不是中性变量
