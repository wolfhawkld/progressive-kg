# 训练调试

## 概念

训练调试是诊断和解决训练问题的过程，是深度学习实践的核心技能。

## 常见问题诊断

### 过拟合
- **症状**: 训练损失持续下降，验证损失上升
- **原因**: 模型太复杂、数据太少、训练太久
- **解决**: 正则化、Dropout、数据增强、早停

### 欠拟合
- **症状**: 训练损失和验证损失都很高
- **原因**: 模型太简单、特征不足、训练不够
- **解决**: 增加模型容量、添加特征、延长训练

### 梯度问题
- **梯度消失**: 浅层参数不更新 → 使用ReLU、残差连接
- **梯度爆炸**: 梯度变成NaN → 梯度裁剪、检查初始化

### 训练不收敛
- 学习率过大或过小
- 数据归一化问题
- 网络结构问题

## 调试工具

### 可视化
- 损失曲线 (TensorBoard)
- 梯度分布
- 激活分布
- 权重分布

### 梯度检查
```python
for name, param in model.named_parameters():
    print(f"{name}: grad_norm = {param.grad.norm()}")
```

### 数值检查
- 检查NaN/Inf
- 监控数值范围

## 调试流程

1. **先过拟合小数据集** - 确保模型有能力学习
2. **逐步扩大数据** - 观察泛化能力
3. **调整超参数** - 学习率、正则化等
4. **监控关键指标** - 损失、准确率、梯度
