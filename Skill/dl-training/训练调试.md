# 训练调试

> 用最小可复现实验逐层排除数据、目标、前向、梯度、优化器、数值与分布式状态错误。

## 总原则

一次只改变一个可观测因素，并先证明最小系统正确，再增加数据、并行、混合精度和正则化。训练曲线只是症状，不能直接把“验证差”归因为过拟合或把 NaN 归因为学习率。

- 优先寻找能推翻当前假设的最小实验。
- 每次复杂化都保留上一阶段可比较的短程基线。

## 排查顺序

顺序从实验身份和数据开始，再进入前向、反向与并行；越早验证的层越接近问题源头。

### 1. 固定实验身份

记录代码提交、配置、数据版本、tokenizer/预处理、依赖、硬件、随机种子和 checkpoint 来源。确认日志中的配置正是运行时配置，而不是默认值副本。

### 2. 检查数据与目标

- 可视化原始样本、增强后样本和标签。
- 检查 split 重复、泄露、类别分布、mask、ignore index 和 padding。
- 用简单规则或小基线确认指标方向、单位和 label mapping。
- 对每个 batch 检查 dtype、shape、范围和非有限值。

### 3. 检查前向与损失

- 在未训练模型上估算合理的初始 loss，例如均匀 $K$ 类交叉熵约为 $\log K$。
- 关闭 dropout/增强等随机性，验证同输入是否得到预期的一致结果。
- 检查 logits 与 loss API 是否重复 sigmoid/softmax，reduction 是否符合全局 batch 语义。

### 4. 过拟合极小数据

选一个小 batch，关闭大多数增强和正则化，尝试把训练损失降到任务允许的低值。失败说明容量、目标、梯度或实现仍有问题；成功只证明可拟合该小样本，不证明泛化或完整 pipeline 正确。

### 5. 检查反向与更新

- 确认预期参数 `requires_grad=True` 且 `grad` 非空。
- 记录分层 gradient/parameter/update norm 和非有限比例。
- 在小模型上做有限差分 gradient check。
- 验证 `zero_grad → backward → clip/unscale → step → scheduler` 顺序。
- 比较一步更新前后的参数，确认优化器确实作用于正确参数组。

### 6. 逐项恢复复杂度

按数据量、增强、正则化、AMP、梯度累积、编译和分布式逐项打开。每一步保留上一阶段的短程对照，出现偏差时可定位到最近变化。

## 症状不是结论

| 症状 | 候选原因 |
|---|---|
| 训练和验证都差 | 实现/标签错误、容量不足、优化失败、指标错误 |
| 训练好验证差 | 泄露后的反向偏差、分布偏移、过拟合、评估模式错误 |
| NaN/Inf | 非法数学操作、AMP 溢出、数据异常、学习率/损失尺度 |
| 梯度为 0/None | 饱和/断图、冻结参数、mask、未参与 loss |
| 恢复后轨迹跳变 | optimizer/scheduler/scaler/RNG/采样位置未恢复 |
| 多卡与单卡不一致 | loss reduction、数据重复、同步、BatchNorm 或随机性 |

## 最小监控集

- train/validation loss 与目标指标；
- learning rate、global step、有效 batch；
- gradient norm、update-to-weight ratio、NaN/Inf；
- 吞吐、数据等待、峰值显存；
- 每个参数组的实际超参数；
- 代表样本的预测和错误切片。

日志本身也要测试：单位、聚合窗口和 rank 去重错误会制造假问题。

## 关系网络

- 诊断 [[梯度消失与梯度爆炸]] — 先区分断图、冻结与真正尺度问题
- 诊断 [[损失函数]] — logits、标签、mask 和 reduction 是高频错误源
- 诊断 [[学习率调度]] — 检查真实 step 口径和恢复连续性
- 诊断 [[混合精度训练]] — scaler、非有限值与 skipped step 需可见
- 诊断 [[分布式训练]] — 先单卡复现，再验证同步和 shard
- 连接 [[超参数调优]] — 只有实现可信后，搜索结果才有意义
