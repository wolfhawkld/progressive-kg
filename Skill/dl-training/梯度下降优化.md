# 梯度下降优化

> 根据目标函数的局部梯度及优化器状态迭代更新参数，在给定预算内寻找较低损失解。

## 核心边界

最基本的更新为：

$$
\theta_{t+1}=\theta_t-\eta_t\nabla L(\theta_t)
$$

负梯度是当前点欧氏几何下最陡的局部下降方向。有限步长、随机 mini-batch 和非凸目标意味着训练不保证单调下降，也不保证找到全局最优。

## Batch、SGD 与 Mini-batch

| 形式 | 梯度来源 | 权衡 |
|---|---|---|
| Full-batch GD | 完整训练集 | 梯度确定但每步昂贵 |
| SGD | 单样本 | 噪声大、更新频繁 |
| Mini-batch SGD | 小批平均 | 适配矩阵硬件，是深度学习常见形式 |

梯度累积可近似更大 batch 的平均梯度，但 dropout、BatchNorm、数据增强、损失 reduction 和优化器 step 时机可能破坏严格等价。

## 主要优化器家族

优化器家族的核心差异是是否累积历史方向、如何预条件梯度，以及怎样处理权重衰减。

### SGD + Momentum

历史梯度状态可减少震荡并加快某些方向的推进，详见 [[动量]]。它通常需要认真调学习率与 schedule，但在一些视觉任务上具有良好泛化经验。

### AdaGrad / RMSprop

AdaGrad 累积梯度平方，为频繁更新坐标降低有效学习率；RMSprop 改用指数移动平均，避免分母只增不减。它们是否“解决”学习率衰减取决于数据和超参数。

### Adam / AdamW

Adam 结合一阶矩和未中心化二阶矩 EMA，并作偏差修正。它对稀疏或尺度差异梯度常很实用，但仍可能对学习率、$\beta$、epsilon 和 batch 很敏感。

AdamW 将 weight decay 与自适应梯度更新解耦。它不是在所有任务上都比 Adam/SGD 泛化更好，只是使衰减语义更清晰。

## 选择流程

1. 先按成熟基线选择 SGD-momentum 或 AdamW。
2. 对齐 loss reduction、有效 batch、梯度累积和 clipping。
3. 优先搜索学习率及 schedule，再调 weight decay、momentum/$\beta$。
4. 比较相同数据、token/epoch、wall time 和计算预算下的验证指标。
5. 保存 optimizer、scheduler、scaler 和随机数状态以实现可比恢复。

“Transformer 必选 AdamW”“CNN 最佳一定是 SGD”都只是经验起点。模型规模、归一化、数据和训练预算会改变结果。

## 失败诊断

| 症状 | 先检查 |
|---|---|
| loss 立刻发散 | 学习率、loss reduction、数据尺度、混合精度溢出 |
| loss 平坦 | 梯度是否存在、学习率是否过小、参数是否被冻结 |
| 训练好验证差 | 数据划分、增强、正则化与分布偏移 |
| 恢复后轨迹跳变 | 是否完整恢复优化器和调度器状态 |
