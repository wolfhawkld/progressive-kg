# 混合精度训练

> 按算子选择 FP16/BF16/FP32 等精度，并用缩放与高精度状态平衡吞吐、显存和数值误差。

## 核心边界

自动混合精度（AMP）不会把所有张量统一转成半精度。矩阵乘/卷积等算子可在低精度运行，归约或数值敏感算子可保留 FP32，参数和优化器状态也可能使用不同 dtype。

它可能提升吞吐和降低部分显存，但收益取决于硬件、shape、kernel、通信和内存瓶颈；模型质量也必须实测，不能承诺“完全不损失精度”。

## FP16 与 BF16

| 格式 | exponent bits | fraction bits | 主要特点 |
|---|---:|---:|---|
| FP32 | 8 | 23 | 范围和有效精度较高 |
| FP16 | 5 | 10 | 尾数多于 BF16，但动态范围窄 |
| BF16 | 8 | 7 | 动态范围接近 FP32，但舍入更粗 |

BF16 较少因范围导致梯度下溢，因此通常不需要 FP16 式 loss scaling；这不是数值永远稳定的保证，累加、softmax、归一化和自定义 kernel 仍可能需要高精度路径。

## Gradient Scaling

FP16 中，小梯度可能在表示时变为 0。动态 scaler 先放大 loss，反向后检查非有限梯度，再在 optimizer step 前取消缩放：

```text
scaled_loss = scale · loss
scaled_loss.backward()
unscale gradients
if finite: optimizer.step()
update scale
```

需要 gradient clipping 时，应先 `unscale_` 再裁剪。scaler 可能降低到 1 以下，不能假设 scale 始终增大或大于 1。

## 当前 PyTorch AMP 骨架

```python
import torch

scaler = torch.amp.GradScaler("cuda")

for inputs, targets in loader:
    optimizer.zero_grad(set_to_none=True)
    with torch.autocast(device_type="cuda", dtype=torch.float16):
        outputs = model(inputs)
        loss = loss_fn(outputs, targets)

    scaler.scale(loss).backward()
    scaler.unscale_(optimizer)
    torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm)
    scaler.step(optimizer)
    scaler.update()
```

`torch.cuda.amp.*` 已被官方标为弃用入口；具体设备和 dtype 应按当前 PyTorch 文档配置。BF16 常用 autocast 后直接 backward/step，但是否使用 scaler 要按设备与模型验证。

## 验证步骤

1. 固定随机性，用 FP32 跑短程基线。
2. 比较同一 batch 的 loss、关键激活、梯度范数与非有限比例。
3. 记录 scaler 变化和 skipped steps。
4. 测量峰值显存、step time 与最终验证指标，而不是套用“2–3 倍”速度表。
5. 对自定义 autograd、归一化、归约和指数/对数算子做专门检查。

低精度权重、master weights 和 optimizer state 的存储策略由优化器与框架实现决定，不能把“计算 FP16、全部关键变量 FP32”当成唯一布局。

## 关系网络

- 影响 [[梯度下降优化]] — 非有限梯度会跳过 step，scaler 状态必须随 checkpoint 保存
- 关联 [[梯度消失与梯度爆炸]] — 数值下溢/溢出与模型 Jacobian 问题相关但不同
- 关联 [[归一化层]] — 归约统计常需要更稳健的累加精度
- 组合 [[分布式训练]] — dtype 可减少通信量，但各 rank 的 scaler/溢出决策需一致
- 对比 [[模型量化]] — AMP 面向训练算子精度，部署量化通常还引入离散 scale/zero-point
