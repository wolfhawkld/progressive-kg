# 混合精度训练

## 概念

混合精度训练使用低精度（FP16/BF16）进行计算，高精度（FP32）存储关键变量，在不损失模型精度的情况下大幅提升训练速度和降低显存占用。

## 核心技术

### 精度对比

| 精度 | 指数位 | 尾数位 | 范围 | 精度 |
|-----|--------|--------|------|------|
| FP32 | 8 | 23 | ±3.4e38 | 高 |
| FP16 | 5 | 10 | ±65504 | 中 |
| BF16 | 8 | 7 | ±3.4e38 | 低 |

### FP16的问题

- **溢出**: 梯度值 > 65504 → Inf
- **下溢**: 梯度值太小 → 0
- **精度损失**: 小梯度四舍五入为0

### Loss Scaling

```
loss_scaled = loss * scale
grad_scaled = ∂loss_scaled / ∂θ
grad = grad_scaled / scale
```

- 放大loss，防止梯度下溢
- 权重更新前缩放回来

**动态缩放**:
- 初始scale较大
- 检测溢出则减小scale
- 稳定训练则增大scale

### FP32 Master Weights

```
计算: FP16
存储: FP32副本
更新: FP32 → FP16
```

- FP32存储权重主副本
- 避免精度累积误差

## BF16的优势

- 与FP32相同的动态范围
- 不需要Loss Scaling
- Ampere架构（A100）及以上支持
- **应用**: 大模型训练首选

## 实现方式

### PyTorch AMP (Automatic Mixed Precision)

```python
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for data, target in dataloader:
    optimizer.zero_grad()
    with autocast():
        output = model(data)
        loss = criterion(output, target)
    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()
```

### BF16模式

```python
with autocast(dtype=torch.bfloat16):
    output = model(data)
# 不需要GradScaler
```

## 性能提升

| 指标 | FP32 | FP16 | BF16 |
|-----|------|------|------|
| 显存占用 | 1x | ~0.5x | ~0.5x |
| 训练速度 | 1x | 2-3x | 2-3x |
| 数值稳定性 | 高 | 中 | 高 |
| 硬件要求 | 通用 | Volta+ | Ampere+ |

## 注意事项

### 适用场景
- 现代GPU（Volta架构及以上）
- 大模型训练（显存瓶颈）

### 不适用场景
- 旧硬件（不支持Tensor Core）
- 对数值精度敏感的任务
- 某些归一化操作（可能溢出）

### 常见问题
- **NaN损失**: scale过大或模型有问题
- **精度下降**: 检查关键层是否需要FP32
- **硬件不支持**: 回退FP32或升级硬件
