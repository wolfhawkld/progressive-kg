# 模型量化

> 用低比特数值和缩放参数近似权重、激活或缓存，以交换模型体积、带宽、速度与误差。

## 核心公式

一种仿射整数映射为：

$$
q=\operatorname{clip}(\operatorname{round}(x/s)+z,q_{min},q_{max}),
\qquad
\hat x=s(q-z)
$$

$s$ 是 scale，$z$ 是 zero-point。量化误差来自舍入、截断和共享 scale；对称量化常令 $z=0$，非对称量化可更好利用偏移范围但实现更复杂。

## 必须先说清的维度

| 维度 | 选择示例 |
|---|---|
| 对象 | weights、activations、KV cache、optimizer state |
| 数值格式 | INT8/INT4、FP8、其他 codebook/非均匀格式 |
| 粒度 | per-tensor、per-channel、per-group、per-token |
| 时机 | static calibration、dynamic quantization、QAT |
| kernel | 真正低比特计算，或仅低比特存储后反量化计算 |

只说“INT4 模型”不足以推断速度、内存或精度。

## PTQ 与 QAT

PTQ 在训练后拟合量化参数，QAT 则让训练过程显式适应量化误差；二者成本和可恢复精度不同。

### Post-Training Quantization

PTQ 在基础模型训练完成后估计 scale/zero-point，并可做权重重构或校准优化。它通常不做完整重训，但仍可能需要代表性 calibration 数据和少量优化步骤。

- GPTQ 用近似二阶信息逐层量化权重。
- AWQ 利用激活统计保护重要权重通道。
- LLM.int8() 把异常值相关计算保留在较高精度路径。

这些名称对应具体论文算法和实现版本，不能泛化为所有 INT4/INT8 方案。

### Quantization-Aware Training

QAT 在训练前向中模拟量化与舍入，使参数适应误差。它常能改善激进低比特配置，但训练成本更高，也不保证优于调校充分的 PTQ。

## 体积与速度边界

从 FP16 权重变为 4-bit 时，纯权重 payload 的理论压缩为 4 倍，不是表格中的 `0.125x`（那是相对 FP32 的 1/8）。真实文件还包含 scales、zero-points、分组元数据和未量化层。

推理是否加速取决于：

- 硬件是否支持对应低比特指令；
- kernel 是否避免昂贵的反量化和重排；
- workload 是内存带宽还是计算受限；
- batch、序列长度和矩阵 shape；
- 是否量化 activation/KV，而非仅权重。

因此不存在 INT8 固定 `2–4x` 或 INT4 固定 `2–5%` 精度损失。

## 实施流程

1. 记录未量化模型、tokenizer、评估集和推理 kernel 基线。
2. 明确要优化文件大小、峰值显存、首 token、decode 吞吐还是能耗。
3. 选择层排除规则、group size、校准数据和 clipping 方法。
4. 同时评估通用 loss/perplexity 与目标任务，检查长上下文和少数类。
5. 在目标硬件上测端到端延迟、吞吐和峰值内存。
6. 保存量化配置与运行库版本，避免“同名格式”无法复现。
