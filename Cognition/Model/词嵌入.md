# 词嵌入

## 概念

词嵌入（Word Embedding）将离散的词映射到连续向量空间，使语义相近的词在向量空间中距离较近。是NLP任务的基础表示层。

## 核心思想

```
词 → 索引 → Embedding层 → 稠密向量
```

- **稀疏 → 稠密**: 从one-hot到低维向量
- **语义编码**: 相似词向量相似
- **维度**: 典型128-1024维

## 经典方法

### Word2Vec

**Skip-gram**: 用中心词预测上下文
```
最大化: P(context | center_word)
```

**CBOW**: 用上下文预测中心词
```
最大化: P(center_word | context)
```

**关键技术**:
- 负采样 (Negative Sampling)
- 层次Softmax

### GloVe (Global Vectors)

```
利用词共现矩阵的全局统计信息
目标: w_i · w_j + b_i + b_j = log(X_ij)
```

- 结合全局统计和局部上下文
- 词向量具有可解释性（类比关系）

### FastText

- 字符级n-gram组合
- 能处理未登录词(OOV)
- 适合形态丰富语言

## 上下文嵌入 (Contextual Embedding)

静态嵌入无法处理多义词。上下文嵌入根据上下文动态生成词向量。

### ELMo

- 双向LSTM
- 词向量是各层输出的加权组合

### BERT

- 双向Transformer编码器
- 词向量由上下文完全决定
- BPE/WordPiece分词

### GPT

- 单向Transformer解码器
- 自回归方式生成表示

## 子词分词

### BPE (Byte Pair Encoding)

- 迭代合并高频字符对
- 平衡词表大小和覆盖率
- 处理OOV问题

### WordPiece

- 基于似然的合并策略
- BERT使用

### SentencePiece

- 语言无关的分词
- 将文本视为字符流
- 支持BPE和Unigram算法

## 嵌入质量评估

### 内在评估

- **词类比**: king - man + woman ≈ queen
- **相似度**: 与人工标注的相关性

### 外在评估

- 下游任务性能
