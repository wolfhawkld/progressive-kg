# 词嵌入

> 将离散词或 token 映射为稠密向量，使训练目标学到的分布规律可用于相似度与下游模型。

## 核心边界

Embedding table 本质上按 token ID 查找可训练向量：

```text
文本 → tokenizer → token IDs → embedding table → vectors
```

语义或句法关系不是查表操作自动保证的，而是由语料、上下文窗口、训练目标和模型归纳偏置塑造。向量“距离近”还依赖所选相似度、归一化和表示空间各向异性。

## 静态词嵌入

静态方法给每个词类型一个固定向量，因此同一个多义词在不同句子中仍共享表示。

### Word2Vec

- Skip-gram 用中心词预测上下文；CBOW 用上下文预测中心词。
- Negative sampling 和 hierarchical softmax 是降低完整词表归一化成本的不同训练方案。
- 常见类比关系是数据与目标产生的规律，不是所有词对上的精确线性定律。

### GloVe

GloVe 在全局共现统计上拟合加权回归目标，简化形式为：

$$
\sum_{i,j}f(X_{ij})
(w_i^T\tilde w_j+b_i+\tilde b_j-\log X_{ij})^2
$$

不能省略权重函数和独立上下文向量后把剩余等式当作模型完整定义。

### FastText

FastText 把词表示构造成字符 n-gram 向量之和，因此能为由已知字符 n-gram 组成的未登录字符串生成向量。它改善形态共享，但不保证任意 OOV 词都获得正确语义。

## 上下文化表示

BERT、GPT 等模型的中间 hidden state 同时受 token 身份、上下文、位置、参数和层深影响。它不是“完全由上下文决定”，也不应把任意 token hidden state 直接当成高质量句向量。

| 模型类型 | 上下文可见性 | 表示特点 |
|---|---|---|
| ELMo | 双向 LSTM 语言模型 | 组合不同层的上下文化词表示 |
| BERT 类 Encoder | 双向掩码上下文 | 适合理解类任务，具体 pooling 需设计 |
| GPT 类 Decoder | 因果左上下文 | 适合自回归建模，当前状态只读允许的过去位置 |

## Tokenizer 与嵌入要分开

BPE、WordPiece、Unigram/SentencePiece 决定字符串如何映射到 token ID；embedding 决定 ID 的向量参数。子词减少封闭词表 OOV，但 byte/character 覆盖、归一化和特殊 token 规则仍会影响可逆性与跨语言表现。

- tokenizer 变更会改变 ID 序列，不能直接复用未对齐的 embedding table。
- embedding 训练目标变更也会改变几何，即使 tokenizer 完全相同。

## 评估与风险

- 内在词相似度和类比测试只覆盖有限语言关系。
- 下游表现更直接，但会混入模型其余部分的贡献。
- 语料偏差会编码进几何空间，近邻结果不等于事实或价值判断。
- 句子检索通常需要专门 pooling/对比训练，并验证点积、余弦或其他距离。

## 关系网络

- 使用 [[内积]] — 点积或余弦常用于表示相似度
- 输入 [[注意力机制]] — Q/K/V 由 token 表示及中间状态投影得到
- 组合 [[位置编码]] — 可加到输入表示，也可通过 RoPE 等方式作用于 Q/K
- 架构 [[Transformer架构]] — token embedding 是常见输入层，也可与输出词表权重共享
- 应用 [[RRF(Reciprocal Rank Fusion)]] — 稠密检索排名可与稀疏排名融合
