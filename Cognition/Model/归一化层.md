# 归一化层

> 按选定维度控制神经网络激活的中心或尺度，常用于改善优化稳定性和数值条件。

## 核心概览

归一化层在指定轴上计算统计量，再对激活做缩放，通常还带可学习的仿射参数。不同方法的关键区别不是名字，而是“对哪些元素共同计算统计量”以及训练和推理是否使用相同统计量。

常见的中心化形式为：

$$
y=\gamma\frac{x-\mu}{\sqrt{\sigma^2+\varepsilon}}+\beta
$$

它常能改善训练稳定性，但不保证在所有模型上加速收敛，也不是防止梯度消失或爆炸的充分条件。

## Batch Normalization

对卷积特征 `[N,C,H,W]`，BN 通常为每个通道在 `N,H,W` 上统计均值和方差。

- 训练时使用当前 mini-batch 统计量，并通常更新运行统计量。
- 推理时通常使用保存的运行均值和方差，具体行为由实现配置决定。
- 小 batch、非独立样本或分布漂移会让统计量更难估计；SyncBN 等方法可跨设备汇总。
- BN 常允许不同的学习率设置并带来一定随机正则化效应，但收益依任务和 batch 构成而异。

BN 可以用于序列数据，只是 padding、时间相关性和 batch 统计依赖使它在许多 Transformer 场景中不如 LN 方便；“不适合变长序列”不是绝对限制。

## Layer Normalization

LN 对每个样本独立地在 `normalized_shape` 指定的特征轴上统计，因此不依赖其他 batch 样本，训练和推理计算形式一致。

- Transformer 中通常沿每个 token 的 hidden dimension 归一化。
- RNN 或图像模型可以选择不同尾部轴，不能统一写成固定 `(C,H,W)`。
- LN 仍受 epsilon、数据类型、归一化位置和残差布局影响。

## Instance 与 Group Normalization

| 方法 | 以 `[N,C,H,W]` 为例的统计范围 | 常见用途 |
|---|---|---|
| InstanceNorm | 每个样本、每个通道的 `H,W` | 风格迁移和图像生成 |
| GroupNorm | 每个样本、每个通道组的 `C/G,H,W` | 小 batch 的视觉模型 |

当 GroupNorm 的组数为 1 或接近通道数时，它分别接近某些 LayerNorm 或 InstanceNorm 配置，但仿射参数和归一化轴仍应按具体实现核对。

## RMSNorm

RMSNorm 不减均值，只按均方根缩放：

$$
y=\gamma\frac{x}{\sqrt{\operatorname{mean}(x^2)+\varepsilon}}
$$

它比 LN 少做中心化，在一些架构中能降低开销并取得相近效果；“效果相当”是经验结果，不是对任意模型的等价性证明。

## Transformer 中的位置

```text
Post-Norm: x = LN(x + Sublayer(x))
Pre-Norm:  x = x + Sublayer(LN(x))
```

Post-Norm 是原始 Transformer 的布局；Pre-Norm 常使深层网络更容易优化，但也会改变残差流尺度与最终归一化需求。现代模型还会使用 RMSNorm、NormFormer、sandwich norm 等变体，因此不存在唯一“标配”布局。

## 选择检查表

| 条件 | 优先考虑 |
|---|---|
| batch 大且统计稳定的 CNN | BatchNorm |
| batch 小或每个样本应独立处理 | GroupNorm / LayerNorm |
| Transformer token 表示 | LayerNorm / RMSNorm，结合残差布局评估 |
| 风格统计需要逐实例控制 | InstanceNorm |

最终应比较训练稳定性、验证指标、推理模式和混合精度误差，而不是只按模型类别套用。

## 关系网络

- 关联 [[标准化]] — 都使用中心和尺度统计，但网络层的统计轴与运行方式更复杂
- 关联 [[梯度消失与梯度爆炸]] — 可改善数值条件，但不能单独保证梯度有界
- 组合 [[残差连接]] — Pre-Norm/Post-Norm 决定归一化与捷径的相对位置
- 应用 [[Transformer架构]] — Transformer 常使用 LN 或 RMSNorm
- 关联 [[混合精度训练]] — 方差归约和 epsilon 对低精度数值稳定性敏感
