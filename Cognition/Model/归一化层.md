# 归一化层

> 归一化层将中间层输出标准化，稳定梯度流动，加速训练收敛。是现代深度网络的核心组件。

---

主要类型

### Batch Normalization (BN)

```
y = γ * (x - μ_batch) / σ_batch + β
```

- **计算维度**: 对每个通道，在batch和空间维度上归一化
- **特点**: 依赖batch统计量，训练/推理行为不同
- **推理**: 使用训练时的滑动平均统计量
- **适用**: 图像CNN，batch较大时效果好

**优点**:
- 加速收敛
- 允许更大学习率
- 一定的正则化效果

**缺点**:
- 小batch不稳定
- 训练/推理不一致
- 不适合序列长度变化的任务

### Layer Normalization (LN)

```
y = γ * (x - μ_layer) / σ_layer + β
```

- **计算维度**: 对每个样本，在特征维度上归一化
- **特点**: 不依赖batch，每个样本独立
- **适用**: RNN、Transformer、NLP任务

**优点**:
- 不依赖batch size
- 训练/推理一致
- 适合变长序列

### Instance Normalization (IN)

```
y = γ * (x - μ_instance) / σ_instance + β
```

- **计算维度**: 对每个样本的每个通道独立归一化
- **适用**: 风格迁移、图像生成

### Group Normalization (GN)

```
y = γ * (x - μ_group) / σ_group + β
```

- **计算维度**: 将通道分组，组内归一化
- **特点**: 介于LN和IN之间
- **适用**: 小batch场景、目标检测

### RMSNorm

```
y = x / RMS(x) * γ
where RMS(x) = sqrt(mean(x²))
```

- **简化**: 去掉均值中心化，只做缩放
- **优点**: 计算更快，效果相当
- **应用**: LLaMA等大语言模型

## 各归一化对比

| 类型 | 归一化维度 | batch依赖 | 适用场景 |
|-----|-----------|----------|---------|
| BatchNorm | (N, H, W) | 是 | 图像CNN，大batch |
| LayerNorm | (C, H, W) | 否 | Transformer，NLP |
| InstanceNorm | (H, W) | 否 | 风格迁移 |
| GroupNorm | (G, C/G, H, W) | 否 | 小batch |
| RMSNorm | (C) | 否 | LLM |

*N: batch size, C: channels, H/W: spatial dims, G: groups*

## Transformer中的归一化

### Pre-Norm vs Post-Norm

**Post-Norm** (原始Transformer):
```
x = LayerNorm(x + Sublayer(x))
```

**Pre-Norm** (现代选择):
```
x = x + Sublayer(LayerNorm(x))
```

Pre-Norm优势:
- 训练更稳定
- 梯度流动更顺畅
- 适合深层网络
