# 在线学习

> 模型按样本或小批数据持续更新的学习范式，适合数据流、增量适应与受限存储场景。

在线学习（Online Learning）按样本或小批数据顺序更新模型。它关注“何时以及如何更新”，并不等同于数据必须逐条到达，也不意味着离线训练必须一次把全部数据装入内存。

核心对比

| 特点 | 批量学习 (Batch Learning) | 在线学习 (Online Learning) |
|------|---------------------------|----------------------------|
| 数据 | 可分批从磁盘读取，但通常可反复访问训练集 | 按到达顺序消费，历史数据可能不可重放 |
| 训练 | 在固定数据快照上优化，部署后可定期重训 | 每批数据到达后增量更新，可持续部署 |
| 存储 | 可保存完整数据集或外部读取 | 算法可只保留当前批和必要状态，也可配合回放缓冲区 |
| 场景 | 静态数据集 | 数据流、实时变化 |

## 工作原理

```python
# 批量学习：一次性训练
model.fit(X_all, y_all)
model.predict(x_new)  # 模型不再变化

# 在线学习：逐个更新
model.partial_fit(x1, y1)  # 第一条数据
model.partial_fit(x2, y2)  # 第二条数据
model.partial_fit(x3, y3)  # 持续...
```

## 典型应用场景

| 场景 | 说明 |
|------|------|
| 推荐系统 | 用户行为实时变化，模型持续调整 |
| 广告点击预测 | 新数据源源不断 |
| 金融风控 | 欺诈手段持续演变 |
| 股票预测 | 市场动态变化 |

## 内存资源分析

在线学习**省的是数据内存，不是权重内存**：

```
批量学习内存占用：
├── 模型权重 (50MB)
├── 全部数据 (5GB)        ← 在线学习不需要
└── 中间梯度 (500MB)

在线学习内存占用：
├── 模型权重 (50MB)       ← 权重仍需加载
├── 当前batch数据 (1MB)
└── 单步梯度 (50MB)
```

**关键**：在线算法可以降低训练数据驻留内存，但模型权重、优化器状态、回放样本和中间激活仍需占用资源。模型可以是小型线性模型，也可以是使用 GPU 的深度网络。

## 常用算法

以下算法具有常见的增量更新形式；是否需要 GPU 由模型和吞吐需求决定：

| 算法 | 说明 |
|------|------|
| SGD（随机梯度下降） | 天然在线，每条数据更新一次 |
| Passive-Aggressive Classifier | 在线分类，适合流式数据 |
| Hoeffding Tree | 在线决策树，增量构建 |
| Online K-Means | 在线聚类，中心点逐步调整 |

```python
from sklearn.linear_model import SGDClassifier

model = SGDClassifier()
for batch in data_stream:
    model.partial_fit(batch.X, batch.y, classes=[0, 1])
```

## 与其他学习范式的对比

| 范式 | 数据处理 | 模型更新 |
|------|----------|----------|
| 批量学习 | 全量数据 | 一次性训练 |
| 在线学习 | 流式数据 | 持续增量更新 |
| [[联邦学习]] | 分布式数据 | 本地训练 + 全局聚合 |

## 适用条件

- ✅ 数据持续到达，无法一次性获取
- ✅ 内存受限，无法存储全部数据
- ✅ 需要模型实时适应变化
- ⚠️ 深度模型也可在线更新，但成本、灾难性遗忘和安全回滚更难管理
- ⚠️ 概念漂移正是在线适应的动机之一；剧烈漂移通常还需要检测、重置、重加权或回放，持续更新本身不会自动解决
