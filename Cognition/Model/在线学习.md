# 在线学习

## 概念

在线学习（Online Learning）是一种机器学习范式：模型**逐个或分批接收数据，持续更新**，而不是一次性用所有数据训练。

## 核心对比

| 特点 | 批量学习 (Batch Learning) | 在线学习 (Online Learning) |
|------|---------------------------|----------------------------|
| 数据 | 一次性全部加载 | 逐个/逐批到达 |
| 训练 | 训练一次，部署后不变 | 持续更新，边用边学 |
| 内存 | 需存储全部数据 | 只需当前数据 |
| 场景 | 静态数据集 | 数据流、实时变化 |

## 工作原理

```python
# 批量学习：一次性训练
model.fit(X_all, y_all)
model.predict(x_new)  # 模型不再变化

# 在线学习：逐个更新
model.partial_fit(x1, y1)  # 第一条数据
model.partial_fit(x2, y2)  # 第二条数据
model.partial_fit(x3, y3)  # 持续...
```

## 典型应用场景

| 场景 | 说明 |
|------|------|
| 推荐系统 | 用户行为实时变化，模型持续调整 |
| 广告点击预测 | 新数据源源不断 |
| 金融风控 | 欺诈手段持续演变 |
| 股票预测 | 市场动态变化 |

## 内存资源分析

在线学习**省的是数据内存，不是权重内存**：

```
批量学习内存占用：
├── 模型权重 (50MB)
├── 全部数据 (5GB)        ← 在线学习不需要
└── 中间梯度 (500MB)

在线学习内存占用：
├── 模型权重 (50MB)       ← 权重仍需加载
├── 当前batch数据 (1MB)
└── 单步梯度 (50MB)
```

**关键**：权重必须加载到内存，但模型通常设计得很小（几KB~几MB），CPU 完全够用。

## 常用算法

这些算法天然支持在线学习，**无需 GPU**：

| 算法 | 说明 |
|------|------|
| SGD（随机梯度下降） | 天然在线，每条数据更新一次 |
| Passive-Aggressive Classifier | 在线分类，适合流式数据 |
| Hoeffding Tree | 在线决策树，增量构建 |
| Online K-Means | 在线聚类，中心点逐步调整 |

```python
from sklearn.linear_model import SGDClassifier

model = SGDClassifier()  # 默认就是在线学习
for batch in data_stream:
    model.partial_fit(batch.X, batch.y, classes=[0, 1])
```

## 与其他学习范式的对比

| 范式 | 数据处理 | 模型更新 |
|------|----------|----------|
| 批量学习 | 全量数据 | 一次性训练 |
| 在线学习 | 流式数据 | 持续增量更新 |
| [[联邦学习]] | 分布式数据 | 本地训练 + 全局聚合 |

## 适用条件

- ✅ 数据持续到达，无法一次性获取
- ✅ 内存受限，无法存储全部数据
- ✅ 需要模型实时适应变化
- ❌ 需要深度学习大模型（权重太大）
- ❌ 数据分布剧烈变化（需要重新训练）
