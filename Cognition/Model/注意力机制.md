# 注意力机制（Attention Mechanism）

> 根据 Query 与 Key 的匹配权重聚合 Value，使模型能按当前查询动态组合上下文信息。

## 直觉理解

注意力不是固定地压缩整个输入，而是针对当前 Query 计算一组权重，再从多个 Value 中组合所需信息。搜索类比可以帮助记忆：Query 是检索需求，Key 用来匹配，Value 是被取回并聚合的内容。

这个类比不表示模型像人一样“理解关注点”，也不表示权重较小的 token 一定无关。

## 缩放点积注意力

$$
\operatorname{Attention}(Q,K,V)
=\operatorname{softmax}\left(\frac{QK^T}{\sqrt{d_k}}+M\right)V
$$

| 符号 | 作用 |
|---|---|
| $Q$ | 提出当前查询 |
| $K$ | 与查询计算兼容度 |
| $V$ | 按权重聚合的内容 |
| $M$ | 可选掩码或位置偏置 |

### 为什么除以 $\sqrt{d_k}$

若 Q/K 各维近似独立且方差相当，点积方差会随 $d_k$ 增长。缩放使 logits 的量级更稳定，减少 Softmax 过早饱和；它不是专门用来“防止梯度消失”的通用保证。

### 掩码

- **Padding mask**：忽略填充位置。
- **Causal mask**：阻止位置 $i$ 读取未来位置 $j>i$。
- **结构掩码**：只开放局部、块状、随机或全局连接。

掩码决定“允许看哪里”，Softmax 再在允许位置中归一化。

## 主要类型

类型差异首先来自 Q/K/V 的来源和可见性掩码，多头机制则在每种类型内部并行多组投影。

### Self-Attention

Q/K/V 来自同一组表示。完整双向 self-attention 可连接所有位置；因果 self-attention 只能读取当前位置及过去；局部 self-attention 还会进一步限制窗口。

### Cross-Attention

Q 来自一个序列，K/V 来自另一组表示，常用于 Encoder–Decoder 和多模态信息融合。

### Multi-Head Attention

$$
\operatorname{MultiHead}(Q,K,V)
=\operatorname{Concat}(head_1,\ldots,head_h)W^O
$$

每个头使用独立投影并并行计算。不同头有能力学习不同交互模式，但并不保证每个头都具有清晰、互异或人类可解释的功能。

## 与 RNN、CNN 的边界

| 机制 | 位置间交互 | 时间步并行 | 主要归纳偏置 |
|---|---|---|---|
| RNN | 隐状态递归传递 | 时间步受限 | 顺序与压缩状态 |
| CNN | 固定局部核逐层扩大范围 | 可并行 | 局部性与平移等变 |
| 全局 Attention | 每层可直接连接允许的任意位置 | 训练时通常可并行 | 内容相关的动态连接，位置需另行注入 |

“Attention 擅长长距离依赖”表示路径更直接，不代表它总比 RNN/CNN 学得更好；数据、优化、位置设计和计算预算仍会影响结果。

## 复杂度与变体

以下只比较序列长度相关的注意力核心，不含 Q/K/V 投影和 FFN：

| 类型 | 典型时间 | 显式中间存储 | 边界 |
|---|---:|---:|---|
| 全局标准注意力 | $O(n^2d)$ | 朴素实现 $O(n^2)$ | 所有位置成对交互 |
| 局部窗口注意力 | $O(nwd)$ | $O(nw)$ | $w$ 为窗口；不是所有“稀疏注意力”都同一复杂度 |
| Kernel/Linear Attention | 常可对 $n$ 近似线性 | 依特征映射而定 | 会改写或近似 Softmax，精度和稳定性依方法而异 |
| FlashAttention | $O(n^2d)$ | 不物化完整矩阵，额外存储近似线性 | 保持标准注意力，优化的是 IO 与 kernel |

GQA/MQA 减少 KV 头数和缓存体积，但不自动消除位置维度上的二次交互。滑动窗口、块稀疏和低秩方法则通过改变可见连接或算子来交换速度与表达能力。

## 简化实现

```python
import math
import torch

def scaled_dot_product_attention(q, k, v, allowed=None):
    scores = q @ k.transpose(-2, -1) / math.sqrt(q.shape[-1])
    if allowed is not None:
        scores = scores.masked_fill(~allowed, torch.finfo(scores.dtype).min)
    weights = torch.softmax(scores, dim=-1)
    return weights @ v, weights
```

真实实现还需处理多头形状、dropout、混合精度、全掩码行以及高效 kernel。

## 注意力权重与解释

权重图可以用于观察模型行为和发现异常，但不能直接等同于因果解释：不同权重分布可能产生相近输出，其他层和 Value 内容也会影响结果。需要解释模型时，应把权重可视化与消融、梯度、反事实或因果干预等证据结合。

- 权重回答“本层如何混合 Value”，不直接回答最终预测为何产生。
- 多层、多头和残差会让单张热力图遗漏其他路径。

## 关系网络

- 应用 [[Transformer架构]] — Transformer 用多头注意力作为位置间信息交换模块
- 归一化 [[Softmax]] — 将允许位置上的匹配 logits 转换为聚合权重
- 依赖 [[位置编码]] — 注意力内容匹配本身不能辨别输入顺序
- 优化 [[kv-cache|KV Cache]] — 自回归解码时复用历史 K/V
- 优化 [[flash-attention|FlashAttention]] — 保持算子定义并减少 HBM 交互
- 关联 [[词嵌入]] — Q/K/V 通常由 token 表示线性投影得到
- 关联 [[残差连接]] — Transformer 把注意力子层放在残差块中
