# 注意力机制（Attention Mechanism）

> 深度学习中的核心机制，让模型**动态关注输入的不同部分**，突破RNN的序列依赖限制，是Transformer的核心创新。

---

## 直觉理解

人类阅读时不会逐词平均处理，而是**聚焦关键词**、忽略无关内容。

```
例子："猫追着老鼠跑进了树林"
人类关注：猫 → 鼠 → 树林
忽略：跑进了、着
```

Attention 让神经网络学会同样的选择性聚焦。

---

## 核心公式

```
Attention(Q, K, V) = softmax(QK^T / √d_k) × V
```

| 角色 | 含义 | 类比 |
|------|------|------|
| **Q (Query)** | 查询向量，"我在找什么" | 搜索关键词 |
| **K (Key)** | 键向量，"我是什么特征" | 书的目录 |
| **V (Value)** | 值向量，"我的内容是什么" | 书的内容 |

**计算流程**：
1. Q 与 K 计算相似度 → 得到注意力分数
2. 除以 √d_k 防止梯度消失
3. Softmax 归一化 → 权重分布
4. 权重加权 V → 输出聚焦后的内容

---

## 注意力类型

### Self-Attention（自注意力）

- Q、K、V 都来自同一输入序列
- 每个位置可以关注序列中所有位置
- **特点**：直接连接、并行计算、无时序限制
- **应用**：Transformer编码器、BERT

### Cross-Attention（交叉注意力）

- Q 来自一个序列，K、V 来自另一个序列
- 实现跨序列信息融合
- **应用**：Encoder-Decoder注意力、多模态融合

### Multi-Head Attention（多头注意力）

```
MultiHead(Q,K,V) = Concat(head_1,...,head_h) × W^O
where head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)
```

- 多个注意力头并行计算，关注不同子空间
- 每个头捕捉不同类型的关联
- **应用**：Transformer标配

### Masked Self-Attention

- 通过掩码阻止关注未来位置
- 用于自回归生成（GPT等）
- 实现：注意力矩阵上三角置为 -∞，softmax后为0

## 与 CNN/RNN 的对比

| 特性 | RNN | CNN | Attention |
|------|-----|-----|-----------|
| **长距离依赖** | 差（逐步传递） | 差（局部窗口） | ✅ 直接连接 |
| **并行计算** | ❌ 时序依赖 | ✅ | ✅ |
| **计算复杂度** | O(n) | O(k·n) | O(n²) |
| **记忆能力** | 隐状态 | 无 | 显式权重矩阵 |

---

## 注意力变体

### 稀疏注意力（Sparse Attention）
- 只关注局部窗口或特定位置
- 降低O(n²)复杂度
- **例子**：Longformer、BigBird

### 线性注意力（Linear Attention）
- 通过核函数近似softmax
- 复杂度降至O(n)
- **例子**：Linear Transformer、Performer

### Flash Attention
- IO感知的精确注意力算法
- 显存高效、计算加速
- **应用**：大模型训练标配

### Grouped Query Attention (GQA)
- 多个Q共享同一组K/V
- 减少KV Cache大小
- **应用**：LLaMA、Mistral等大模型

### Sliding Window Attention
- 局部窗口注意力
- **应用**：长文档处理、Mistral

---

## 计算复杂度

| 类型 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 标准注意力 | O(n²d) | O(n²) |
| 稀疏注意力 | O(n·w·d) | O(n·w) |
| 线性注意力 | O(nd²) | O(nd) |
| Flash Attention | O(n²d) | O(n) |

*n: 序列长度, d: 特征维度, w: 窗口大小*

## Python 实现（简化版）

```python
import torch
import torch.nn as nn
import math

class SelfAttention(nn.Module):
    def __init__(self, d_model, d_k):
        super().__init__()
        self.W_q = nn.Linear(d_model, d_k)
        self.W_k = nn.Linear(d_model, d_k)
        self.W_v = nn.Linear(d_model, d_k)
        self.d_k = d_k
    
    def forward(self, x, mask=None):
        Q = self.W_q(x)
        K = self.W_k(x)
        V = self.W_v(x)
        
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        
        attn_weights = torch.softmax(scores, dim=-1)
        output = torch.matmul(attn_weights, V)
        
        return output, attn_weights

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        self.d_k = d_model // num_heads
        self.num_heads = num_heads
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)
    
    def forward(self, x, mask=None):
        batch_size = x.size(0)
        
        Q = self.W_q(x).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        K = self.W_k(x).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        V = self.W_v(x).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        attn_weights = torch.softmax(scores, dim=-1)
        attn_output = torch.matmul(attn_weights, V)
        
        attn_output = attn_output.transpose(1, 2).contiguous().view(batch_size, -1, self.num_heads * self.d_k)
        output = self.W_o(attn_output)
        return output
```
