# 残差连接

> 将变换分支与恒等或投影捷径相加，为深层网络提供直接的信息和梯度传播路径。

残差连接（Residual Connection）把变换分支 $F(x)$ 与捷径分支相加。它让恒等映射容易表示，并为前向信息和反向梯度提供较短路径，显著缓解了极深网络的优化退化；它并不从数学上保证梯度永不消失或爆炸。

核心公式

```
y = F(x) + x
```

- **x**: 输入（恒等映射）
- **F(x)**: 残差函数，网络层学习的目标
- **y**: 输出

## 为什么有效

残差连接的收益可从梯度路径、恒等基线和多路径解释三个角度理解，但这些角度都不是无条件保证。

### 梯度流动

```
∂y/∂x = ∂F(x)/∂x + 1
```

- Jacobian 中出现恒等项，为梯度提供直接的加性路径
- $I+J_F$ 仍可能发生抵消、放大或数值不稳定，因此不存在“梯度至少为 1”的普遍下界

### 优化难度

- 当输入输出维度一致时，令残差分支接近 0 即可近似恒等映射
- 这给优化器提供了方便的基线，但“学习零一定比学习恒等更容易”是经验解释，不是无条件定理
- 网络可以"选择性"学习需要的变化

### 集成效应

- 残差网络可被解释为许多不同长度路径共同贡献
- “指数数量子网络的集成”是一种分析视角，不是与独立训练集成模型严格等价

## 主要变体

残差加法、门控加法和特征拼接都属于跳跃连接家族，但它们的维度与信息保留方式不同。

### 标准残差

```
y = F(x) + x
```

### Pre-activation ResNet

```
y = x + F(Norm(Act(x)))  # BN和ReLU在卷积前
```

- 更好的梯度流动
- 训练更稳定

### 其他跳跃连接

```
y = Concat([x_1, x_2, ..., x_l])
```

- DenseNet 使用拼接而非残差加法；U-Net 常把编码器特征拼接或相加到解码器
- 它们都属于广义 skip connection，但不应都称为残差连接变体

### Highway Network

```
y = T * F(x) + (1 - T) * x  # T为门控
```

- 可学习的残差比例

## 应用场景

| 架构 | 残差形式 |
|-----|---------|
| ResNet | 标准残差 |
| Transformer | Post-Norm/Pre-Norm |
| U-Net | 跨尺度 skip connection，常用拼接 |
| DenseNet | 多层特征拼接，不是残差加法 |
| Transformer | 围绕注意力和 FFN 的加法残差，可用 Pre-Norm 或 Post-Norm |

## Transformer中的残差

```
# Pre-Norm (现代选择)
x = x + Attention(LayerNorm(x))
x = x + FFN(LayerNorm(x))

# Post-Norm (原始)
x = LayerNorm(x + Attention(x))
x = LayerNorm(x + FFN(x))
```

原始 Transformer 和 BERT 采用 Post-Norm；许多后来的深层 Transformer 与大语言模型采用 Pre-Norm 或其他归一化布局。不能把 BERT/GPT 作为统一的 Pre-Norm 类别。

## 注意事项

- **维度匹配**: F(x)和x维度必须一致，否则需要投影
- **加法位置**: 通常在激活函数之前
- **内存开销**: 需要保存输入用于反向传播
