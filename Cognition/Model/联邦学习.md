# 联邦学习

> 多方在原始数据留存本地的前提下协同训练，并交换模型更新或其受保护聚合结果。

联邦学习（Federated Learning）让客户端在本地数据上计算更新，再由协调方聚合共享模型。协议可传输权重差、梯度、统计量和必要元数据，并非只允许一种“模型参数”；即使原始样本不上传，更新也可能泄露信息，因此联邦学习本身不等于隐私保证。

核心思想

```
传统学习：
数据 → 中心服务器 → 训练模型
（隐私风险，数据泄露）

联邦学习：
      数据(本地)    数据(本地)    数据(本地)
         ↓            ↓            ↓
      本地训练      本地训练      本地训练
         ↓            ↓            ↓
      模型更新      模型更新      模型更新
         └────────────┼────────────┘
                      ↓
                 聚合中心（接收模型更新）
                      ↓
                 全局模型
                      ↓
                 分发回各方
```

## 典型应用场景

| 场景 | 参与方 | 数据 |
|------|--------|------|
| 手机输入法预测 | 数百万用户手机 | 每个人的打字记录 |
| 医疗诊断模型 | 多家医院 | 各自的病历数据 |
| 银行风控 | 多家银行 | 各自的交易记录 |
| 跨企业协作 | 不同公司 | 商业机密数据 |

## 工作流程

```
1. 中心初始化模型 → 发给各方
2. 各方本地训练 → 得到模型更新（梯度/权重变化）
3. 各方上传更新 → 中心聚合（加权平均）
4. 中心下发新模型 → 重复步骤2
```

## 聚合方式

最常用的是 **FedAvg（联邦平均）**：

```
全局模型新权重 = Σ (各方权重更新 × 各方数据量权重) / 总数据量
```

数据多的方贡献更大权重。

## 本地参与门槛

本地设备必须能：

| 能力 | 必须 | 说明 |
|------|------|------|
| 加载权重到内存 | ✅ | 如 PyTorch state_dict |
| 本地前向传播 | ✅ | 跑一遍模型，算预测 |
| 本地反向传播 | ✅ | 计算梯度/权重更新 |
| 上传更新的能力 | ✅ | 网络 + 发送数据 |

```python
# 本地收到全局模型
model = SimpleNet()
model.load_state_dict(received_weights)  # 加载权重

# 本地训练
for x, y in local_data:
    model.train_step(x, y)

# 上传协议规定的模型更新，而非原始训练样本
weight_update = model.get_updates()
send_to_server(weight_update)
```

## 资源与模型大小

参与设备必须能执行协议要求的本地计算，但不存在按 10 MB、100 MB 划分设备适用性的通用阈值。可行性取决于权重与优化器状态、峰值激活、量化、训练层数、网络带宽、能耗和在线时长。

跨设备联邦通常受手机或边缘设备约束；跨机构联邦的参与方可能是医院或企业服务器。大模型也可只训练适配器、局部层或压缩更新，但这并不消除通信、隐私和异构性挑战。

## 优点与挑战

| 优点 | 缺点 |
|------|------|
| 减少集中收集原始数据的需要 | 通信开销大（反复传更新） |
| 可配合组织边界和数据治理 | 不自动满足 GDPR 等法规，也不自动防止更新泄露 |
| 利用分散数据价值 | 可能被恶意参与者攻击 |
| 无需中心化数据存储 | 调试、监控复杂 |

## 安全性增强

联邦学习本身不保证绝对安全，通常会结合：

| 技术 | 说明 |
|------|------|
| 差分隐私 | 在更新中加噪声，防止反推原始数据 |
| 安全聚合 | 加密传输，中心只能看到聚合结果 |
| 同态加密 | 在加密数据上计算 |

## 与其他学习范式的对比

| 范式 | 数据位置 | 模型更新 |
|------|----------|----------|
| 批量学习 | 中心化 | 一次性训练 |
| [[在线学习]] | 本地流式 | 本地持续更新 |
| 联邦学习 | 分布式本地 | 本地训练 + 全局聚合 |

## 关键区别

- 传输对象由协议决定，可能是权重、差分、梯度或受保护的聚合份额
- 原始训练样本按设计留在本地，但缓存、日志和攻击面仍需审计
- 隐私通常还需安全聚合、差分隐私、访问控制与威胁建模

## 关系网络

- 应用：[[分布式训练]] — 都涉及多节点协调，但信任边界不同
- 对比：[[在线学习]] — 二者解决的数据组织问题不同
