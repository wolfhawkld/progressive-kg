# 联邦学习

## 概念

联邦学习（Federated Learning）：数据不动，模型动。各方的数据**留在本地**，只交换**模型参数**，协作训练一个共享模型。

## 核心思想

```
传统学习：
数据 → 中心服务器 → 训练模型
（隐私风险，数据泄露）

联邦学习：
      数据(本地)    数据(本地)    数据(本地)
         ↓            ↓            ↓
      本地训练      本地训练      本地训练
         ↓            ↓            ↓
      模型更新      模型更新      模型更新
         └────────────┼────────────┘
                      ↓
                 聚合中心（只收模型参数）
                      ↓
                 全局模型
                      ↓
                 分发回各方
```

## 典型应用场景

| 场景 | 参与方 | 数据 |
|------|--------|------|
| 手机输入法预测 | 数百万用户手机 | 每个人的打字记录 |
| 医疗诊断模型 | 多家医院 | 各自的病历数据 |
| 银行风控 | 多家银行 | 各自的交易记录 |
| 跨企业协作 | 不同公司 | 商业机密数据 |

## 工作流程

```
1. 中心初始化模型 → 发给各方
2. 各方本地训练 → 得到模型更新（梯度/权重变化）
3. 各方上传更新 → 中心聚合（加权平均）
4. 中心下发新模型 → 重复步骤2
```

## 聚合方式

最常用的是 **FedAvg（联邦平均）**：

```
全局模型新权重 = Σ (各方权重更新 × 各方数据量权重) / 总数据量
```

数据多的方贡献更大权重。

## 本地参与门槛

本地设备必须能：

| 能力 | 必须 | 说明 |
|------|------|------|
| 加载权重到内存 | ✅ | 如 PyTorch state_dict |
| 本地前向传播 | ✅ | 跑一遍模型，算预测 |
| 本地反向传播 | ✅ | 计算梯度/权重更新 |
| 上传更新的能力 | ✅ | 网络 + 发送数据 |

```python
# 本地收到全局模型
model = SimpleNet()
model.load_state_dict(received_weights)  # 加载权重

# 本地训练
for x, y in local_data:
    model.train_step(x, y)

# 只上传权重变化，不上传数据
weight_update = model.get_updates()
send_to_server(weight_update)
```

## 模型大小限制

**模型必须足够小**，能在本地设备上加载：

| 模型大小 | 适合设备 |
|----------|----------|
| < 10MB | 手机、IoT |
| 10-100MB | 手机、电脑 |
| 100MB-1GB | 电脑、服务器 |
| > 1GB | 服务器、云端 |

联邦学习常用小型模型，大模型（如 LLM）目前很难在手机端做联邦学习。

## 优点与挑战

| 优点 | 缺点 |
|------|------|
| 数据不出本地，隐私保护 | 通信开销大（反复传模型） |
| 合规（GDPR等） | 各方数据分布可能不均衡 |
| 利用分散数据价值 | 可能被恶意参与者攻击 |
| 无需中心化数据存储 | 调试、监控复杂 |

## 安全性增强

联邦学习本身不保证绝对安全，通常会结合：

| 技术 | 说明 |
|------|------|
| 差分隐私 | 在更新中加噪声，防止反推原始数据 |
| 安全聚合 | 加密传输，中心只能看到聚合结果 |
| 同态加密 | 在加密数据上计算 |

## 与其他学习范式的对比

| 范式 | 数据位置 | 模型更新 |
|------|----------|----------|
| 批量学习 | 中心化 | 一次性训练 |
| [[在线学习]] | 本地流式 | 本地持续更新 |
| 联邦学习 | 分布式本地 | 本地训练 + 全局聚合 |

## 关键区别

- **上传的是权重变化**，不是整个模型（大小相同，但概念上强调"只传变化"）
- **数据始终留在本地**，隐私得到保护
- 模型权重必须能加载到本地设备内存
