# 位置编码

> 为自注意力注入顺序、距离或坐标信息，以区分仅由内容无法区分的排列关系。

不含位置信息的自注意力对输入排列是**等变**的：输入 token 同步重排时，输出也同步重排；若随后做无序聚合，整体表示才可能表现为排列不变。位置编码不必保证每个位置拥有数学上唯一的向量，其目标是让模型能够利用顺序、相对距离或空间坐标。

主要类型

位置可以通过输入向量、注意力偏置或 Q/K 变换注入；绝对与相对信息也可以组合。

### Sinusoidal位置编码 (原始Transformer)

```
PE(pos, 2i) = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))
```

- **特点**: 固定编码，不需要学习
- **优点**: 公式可在训练长度之外继续计算，但模型质量能否外推没有保证
- **原理**: 不同频率的正弦/余弦组合，位置关系可通过线性变换表示

### 可学习位置编码

```
PE = Embedding(max_seq_len, d_model)  # 可训练
```

- **特点**: 位置嵌入作为参数学习
- **优点**: 适应特定任务
- **缺点**: 超出嵌入表范围时需要扩表或插值；即使扩展，质量也取决于训练方式
- **应用**: BERT、GPT-2

### 相对位置编码

**核心思想**: 编码位置间的相对距离，而非绝对位置

**代表方法**:
- **Shaw's RPE**: 在注意力分数中加入相对位置偏置
- **T5 Bias**: 将相对距离分桶，并为每个桶学习注意力偏置
- **ALiBi**: 线性偏置，根据距离线性衰减注意力

```
Attention = QK^T + bias  # bias由相对位置决定
```

### RoPE (Rotary Position Embedding)

**核心思想**: 将位置信息编码为旋转矩阵，作用于Q和K

```
f(x, m) = x * e^(imθ)  # 复数形式旋转
```

- **特点**: 结合绝对和相对位置优点
- **优点**: 
  - 自然表达相对位置
  - 原生 RoPE 的长度外推并不自动可靠，常需频率缩放、插值或长上下文继续训练
  - 无需额外参数
- **应用**: LLaMA、GLM、PaLM
- **详见**: [[RoPE]]

### ALiBi (Attention with Linear Biases)

```
Attention(i, j) = q_i · k_j - m · |i - j|
```

- **特点**: 根据距离线性惩罚注意力分数
- **优点**: 原论文在特定语言模型设置中展示了较好的长度外推；并非所有任务和长度上的固定保证
- **应用**: BLOOM、MPT

## 长度外推对比

| 方法 | 外推能力 | 参数量 | 适用场景 |
|-----|---------|--------|---------|
| Sinusoidal | 可计算到任意位置，质量依赖训练 | 0 | 通用基线 |
| 可学习 | 受嵌入表范围约束 | O(L*d) | 固定或扩表后的长度 |
| RoPE | 原生外推有限，可配合缩放方法 | 0 | 许多 LLM |
| ALiBi | 某些设置下外推较好 | O(H) | 不显式添加位置向量的序列模型 |

*L: 最大长度, d: 维度, H: 注意力头数*

## 1D vs 2D位置编码

**1D序列**:
- 文本、时间序列
- RoPE、ALiBi常用

**2D图像**:
- ViT需要编码空间位置
- 方法: 
  - 两个1D编码拼接
  - 2D正弦编码
  - 可学习2D嵌入
