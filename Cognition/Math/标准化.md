# 标准化

> 用训练数据的均值和标准差缩放特征，使其在该数据上的均值为0、标准差为1。

---

## 定义与公式

标准化（Z-score）通过减均值、除标准差，把不同量纲的数据映射到统一统计坐标系。核心不是"拍扁到[0,1]"，而是用"偏离均值几个标准差"来衡量——保留相对间距，丢弃绝对量纲。

- **公式**：x' = (x - μ) / σ
- **结果**：均值0，标准差1，分布形态不变
- **线性变换**：不改变数据间的相对顺序和距离比例
- **无界输出**：不限定在[0,1]，理论上可取任意实数

### Z-score 推导

给定特征列 X，计算 μ = mean(X)，σ = std(X)，变换后 X' 满足：

```
E[X'] = E[(X-μ)/σ] = (E[X]-μ)/σ = 0
Var[X'] = Var[(X-μ)/σ] = Var[X]/σ² = 1
```

直观理解：成绩85分比平均高1.2σ，身高175cm也比平均高1.2σ——两个维度的"相对位置"相同。

### 其他标准化方法

| 方法 | 公式 | 特点 |
|---|---|---|
| **Z-score** | (x-μ)/σ | 保留分布形态，但均值和标准差会受异常值影响 |
| **Robust Scaler** | (x-median)/IQR | 使用稳健统计量，通常比 Z-score 更不易受异常值影响 |
| **MaxAbs Scaler** | x/max(|x|) | 保持稀疏性，适合稀疏数据 |

### 标准化 vs 常见缩放方法

术语并不完全统一：有些资料把 Min–Max 缩放称为“归一化”，另一些资料用 normalization 指把每个样本缩放到单位范数。判断方法时应直接看公式，而不能只看中文名称。

| 维度 | 标准化 (Standardization) | 归一化 (Normalization) |
|---|---|---|
| **公式** | (x-μ)/σ | (x-min)/(max-min) |
| **输出范围** | 无界（通常~[-3,3]） | 严格[0,1] |
| **中心化** | ✅ 以0为中心 | ❌ 不以0为中心 |
| **分布形态** | 仿射变换，不改变偏度等形态 | 仿射变换，同样保留相对位置 |
| **异常值敏感度** | 高：均值和标准差会被影响 | 高：极值直接决定缩放范围 |
| **适用判断** | 希望按标准差表达尺度 | 需要有界区间且训练范围稳定 |

深度学习常受益于尺度受控、近零中心的输入，但具体预处理取决于数据模态、网络结构和预训练约定。图像模型常见固定区间缩放后再按通道标准化；不能简单断言所有场景都偏好同一种方法。

## 机器学习中的应用

基于距离或梯度的ML算法对特征尺度敏感，不标准化会导致大数值特征主导计算。树模型是例外——按阈值分裂只看相对顺序。

- **距离失真**：KNN/K-means/SVM中大数值特征主导距离
- **梯度不均**：尺度差异会使优化问题条件数变差；逐列标准化通常改善条件数，但特征相关性仍可能保留椭圆形等高线
- **正则化不公平**：L1/L2对大数值特征和小数值特征施加相同惩罚

### 距离/相似度敏感模型

| 算法 | 为什么需要标准化 |
|---|---|
| **KNN** | 欧氏距离依赖每个特征尺度，未标准化=大尺度特征主导 |
| **K-Means** | 聚类中心更新逻辑与KNN一致 |
| **SVM (RBF核)** | 核函数计算依赖距离，量纲不同导致决策边界偏移 |
| **PCA** | PCA 对尺度敏感；当不同量纲应被同等对待时通常先标准化 |

### 梯度优化模型

- **线性/逻辑回归**：理论上收敛不受影响，但收敛速度严重依赖特征尺度
- **Lasso/Ridge**：正则化惩罚假设所有特征"同等重要"，但看的是数值绝对值——不标准化就是对正确的事施加错误的惩罚

### 不需要标准化的模型

**树模型（决策树、随机森林、XGBoost、LightGBM）**通常不需要为尺度而标准化，因为单调仿射缩放不改变候选切分的顺序。直方图分箱、数值精度和其他流水线组件仍可能造成实现层面的差异。

## 深度学习中的应用

标准化思想在DL中从数据预处理发展为网络结构的一部分——BatchNorm/LayerNorm等"归一化层"数学上做的就是Z-score + 可学习参数。

- **输入标准化**：训练前对输入做Z-score，使激活函数工作在敏感区
- **内部标准化**：BN/LN在每层之后做标准化，稳定训练
- **与初始化的关系**：He/Xavier 初始化用于控制层间方差；合理的输入尺度有助于满足其近似假设，但“未标准化就失效”并不成立

### 输入层标准化

不做标准化的后果：sigmoid/tanh工作区间严重偏移；梯度在各维度量级不一致，某些参数更新过快/过慢。

### Batch Normalization (BN)

```
x̂ = (x - μ_batch) / σ_batch  →  y = γ·x̂ + β
```

- **核心**：在网络内部每层后对mini-batch做Z-score，再通过可学习的γ（缩放）和β（偏移）恢复表达能力
- **γ,β的作用**：恢复可学习的仿射自由度，使该层能够选择合适的尺度和偏移，必要时也可表示恒等映射
- **训练影响**：常能改善优化稳定性并允许不同的学习率设置，但收益大小依赖架构、批大小和任务

### Layer Normalization (LN) — Transformer标配

```
x̂_i = (x_i - μ_features) / σ_features
```

- **与BN的区别**：BN沿batch维度标准化，LN沿feature维度标准化
- **为什么Transformer用LN**：NLP中batch内序列长度不同→BN统计量不稳定；LN对每个样本独立做，天然适配变长序列；推理时不依赖batch统计量

### Normalization 变体总览

| 方法 | 标准化维度 | 典型场景 |
|---|---|---|
| **Batch Norm** | batch + 空间 | CNN标配 |
| **Layer Norm** | 通道 + 空间 | Transformer/RNN |
| **Instance Norm** | 单样本单通道 | 风格迁移 |
| **Group Norm** | 分组通道 + 空间 | 小batch场景 |
| **RMS Norm** | 省去中心化，只缩放 | LLaMA等大模型 |

这些方法都在控制激活尺度，但统计量和目标不同。例如 RMSNorm 不减均值，因此不能都视为严格的 Z-score，也不保证输出服从某个固定分布。

## 强化学习中的应用

强化学习中的观测、回报和优势尺度常随策略与环境变化，运行统计量因而是常见的稳定化工具，但是否使用、在哪一层使用取决于算法和环境。

- **数据来源**：在线交互，分布持续变化（vs ML的静态数据集）
- **统计量**：需要running mean/std动态更新（vs ML一次性计算）
- **风险**：尺度失衡可能导致优化不稳定；标准化也可能因统计量滞后或泄露非平稳信息而引入偏差

### 状态标准化 (State Normalization)

```
s' = (s - μ_running) / σ_running
```

- 使用运行时均值和标准差，消除不同状态维度间的量纲差异（位置vs速度vs角度）
- 在连续控制等任务中很常见，但不是 PPO、A3C 等算法定义上的必需步骤

### 奖励标准化 (Reward Normalization)

```
r' = r / σ_running
```

- 可以缩放即时奖励，也可以按折扣回报的运行方差缩放；不同实现是否减均值并不相同
- 防止reward scale过大导致梯度爆炸
- 也常用固定裁剪：r = clip(r, -1, 1)（如DQN的Atari实验）

### 优势函数标准化 (Advantage Normalization)

许多 PPO 实现会在 mini-batch 或 rollout 内对 advantage 做 Z-score：

```
Â' = (Â - μ_batch) / σ_batch
```

这样通常能控制梯度尺度，但它属于常见实现选择，并不是 PPO 目标函数成立的必要条件，也不应误写成原始论文规定的默认步骤。

## Python 实现

sklearn用于静态数据集；RL需要自定义RunningMeanStd处理流式数据。关键陷阱：测试集必须用训练集的μ和σ，不能自己fit。

### sklearn 方式

```python
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)   # 计算μ,σ并变换
X_test_scaled  = scaler.transform(X_test)         # 用训练集的μ,σ变换
# 注意：测试集不能用 fit_transform！
```

### RL 运行时标准化

```python
import numpy as np

class RunningMeanStd:
    """RL的运行时均值和标准差追踪"""
    def __init__(self, shape=(), epsilon=1e-4):
        self.mean = np.zeros(shape, dtype=np.float64)
        self.var = np.ones(shape, dtype=np.float64)
        self.count = epsilon

    def update(self, x):
        batch_mean = np.mean(x, axis=0)
        batch_var = np.var(x, axis=0)
        batch_count = x.shape[0]
        delta = batch_mean - self.mean
        tot_count = self.count + batch_count
        self.mean += delta * batch_count / tot_count
        m_a = self.var * self.count
        m_b = batch_var * batch_count
        M2 = m_a + m_b + delta**2 * self.count * batch_count / tot_count
        self.var = M2 / tot_count
        self.count = tot_count

    def normalize(self, x):
        return (x - self.mean) / np.sqrt(self.var + 1e-8)
```

## 关系网络

- 关联 [[正交]] — 正交初始化保信号幅度，标准化保分布稳定，两者互补
- 关联 [[主成分分析(PCA)]] — 当特征量纲不宜直接比较时，先标准化再做 PCA
- 对比 [[幂变换]] — 幂变换调整分布形状，标准化只改变中心与尺度
- 关联 [[归一化层]] — BN/LN/IN/GN数学上就是Z-score + 可学习参数
- 关联 [[梯度下降优化]] — 特征缩放常能改善条件数，但不保证等高线变成圆形
- 关联 特征工程（待建） — 标准化属于特征缩放(Feature Scaling)
