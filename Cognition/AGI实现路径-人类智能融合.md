# AGI 实现路径：人类智能融合

> 研究假设：以人类行为理论作先验并通过持续交互学习，可能提高 Agent 的泛化与协作能力。

## 假设陈述

如果把语言行为、意图、决策和协作理论编码为 Agent 的结构先验，再通过真实交互持续修正，这类系统可能比只从静态任务数据拟合的系统更快形成可迁移的人类协作模型。

这是研究方向，不是已证实的 AGI 路径。它至少包含三个尚未证明的跃迁：

1. 可观测行为能否充分约束隐藏意图与认知状态；
2. 行为预测能力能否转化为开放世界规划与自主能力；
3. 更像人类或更懂人类是否真的带来安全、价值对齐和通用智能。

## 两个互补建模轴

以下不是穷尽性的“两条 AGI 路线”，而是可组合的关注轴：

| 轴 | 主要对象 | 可观测目标 | 典型失败 |
|---|---|---|---|
| 世界/动力学建模 | 环境状态、因果与行动后果 | 预测、控制、规划 | 会预测但不了解人的偏好或规范 |
| 人类行为建模 | 语言行动、意图、反馈与协作 | 解释、协商、适应 | 模仿表象、误判意图、复制偏差 |

具身 Agent 可以同时学习环境动力学和人类反馈；两轴不是自底向上/自顶向下的互斥选择。

## 理论框架的作用

Speech Act Theory、意图 taxonomy、行为经济学等可以提供：

- 初始标签空间和可解释假设；
- 数据采集与错误分析的维度；
- 交互策略的先验约束；
- 发现现有分类无法覆盖样本的参照。

它们更像可修订的 ontology 或 prior，不是人类智能的“源代码”或完备 API。理论类别会受文化、语境和任务影响，也可能把连续、多重或策略性意图压进过窄标签。

## 候选学习闭环

```text
行为理论/ontology
        ↓ 初始化可解释假设
交互 Agent → 记录行为、上下文、反馈与不确定性
        ↓
错误切片 / 聚类 / 主动询问 / 人工复核
        ↓
更新模型、类别定义或交互策略
        ↓
跨人群、跨任务和安全评估
```

持续更新只说明系统会适应，不保证单调进步。反馈回路可能造成迎合、偏见放大、分布坍缩和 reward hacking，因此需要版本化、离线回放、回滚和独立评估。

## 可证伪预测

| 预测 | 对照 | 失败判据示例 |
|---|---|---|
| 理论先验提高冷启动样本效率 | 同模型、同预算的数据驱动标签 | 在未见领域上无提升或更差 |
| 交互更新提高跨用户适应 | 静态模型/仅追加数据重训 | 适应收益不超过成本或引入遗忘 |
| 显式意图状态改善协作 | 端到端隐状态 Agent | 任务成功、校准或可纠错性无改善 |
| ontology 可帮助发现新模式 | 无 ontology 的聚类/表示基线 | 只重述既有类别，无法预测外部结果 |

“达到 AGI”过于宽泛，当前阶段应先用样本效率、跨域泛化、协作成功率、校准、纠错成本和安全失败率验证局部主张。

## 对齐边界

从人类行为学习不会天然对齐：人类行为包含冲突价值、欺骗、偏见和情境妥协。行为模仿、意图推断、规范学习和最终价值选择必须分开评估。

更可解释的 taxonomy 也只提供观察窗口；模型可能借助其他内部特征作出同样预测，因此还需要反事实、干预和审计证据。

## 当前最小研究计划

1. 选定一个可衡量的多轮意图任务，而不是直接以 AGI 为目标。
2. 定义理论先验、纯数据基线和混合模型。
3. 在同数据/算力下比较冷启动和未见领域表现。
4. 记录拒答、歧义、多意图和 taxonomy 外样本。
5. 用真实用户纠错评估交互更新，并保留冻结对照组。
6. 只有局部预测反复成立后，再扩大到行动与长期规划。

## 开放问题

1. 哪些行为维度能跨文化和任务稳定迁移？
2. 如何区分“预测人类”与“理解/协作”的增量价值？
3. ontology 何时帮助样本效率，何时限制新模式发现？
4. 如何防止交互闭环把短期满意度误当长期价值？
