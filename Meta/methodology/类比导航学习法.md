# 类比导航学习法

> 先用标注边界的结构类比建立概念地图，再沿双链回到定义、证据和实现细节。

## 方法定位

类比在这里是导航脚手架，不是证明。它把陌生系统中的关系映射到熟悉系统，帮助回答“组件在哪里、相互怎样作用、下一步该点开什么”；精确结论仍由公式、实验、来源和边界条件承担。

本方法对应本库的渐进披露：

```text
L1：一句话定位与类比入口
 ↓
L2：关键机制、比较和适用边界
 ↓
L3：公式、代码、证据、反例与实现
```

## 类比卡片

每个类比都记录五项：

| 字段 | 问题 |
|---|---|
| Target | 要理解的真实概念是什么？ |
| Source | 借用哪个熟悉系统？ |
| Mapping | 哪些关系一一对应？ |
| Inference | 类比帮助推出什么待验证猜想？ |
| Breaks | 哪些地方不能继续类推？ |

如果只能说“它很像某物”却写不出关系映射和失效点，这个类比适合作记忆标签，不适合作推理工具。

## 操作流程

流程从精确定义开始，经结构映射和边界记录，最终必须回到无类比的检索与验证。

### 1. 写精确定义

先用不含类比的一句话定义 target，防止类比反过来替代概念。

### 2. 选择结构相似的 source

优先匹配因果、信息流、约束或变换关系，而不是颜色、形状等表面相似。

### 3. 建立映射表

以“Transformer 像开放式办公室”为例：

| 类比元素 | Target 元素 |
|---|---|
| 人员 | token 位置 |
| 是否听取某人 | 由 Q/K 内容计算的注意力权重 |
| 多个讨论组 | 多个 attention heads |
| 访问限制 | causal/local mask |

### 4. 立即写失效边界

这个类比不能推出：token 有人的意图；每个位置都同等交流；自回归生成完全并行；注意力权重就是解释。记录这些 breaks，类比才不会把旧直觉偷偷带进新领域。

### 5. 连接精确页面

类比卡只保留导航，把细节链接到 [[Transformer架构]]、[[注意力机制]]、[[位置编码]] 等原子概念。发生事实修订时只改精确页面和受影响映射，不复制多份定义。

### 6. 做无类比回忆

关闭类比，尝试回答定义、公式、适用条件和一个反例。若只能复述故事，说明尚未从脚手架过渡到知识。

## 质量检查

- 类比是否只解释一个核心关系，而不是包办整个概念？
- 是否明确“相同点”和“不同点”？
- 能否从 target 的正式定义重新推出结论？
- 是否有至少一个反例阻止过度类推？
- 双链是否指向实际存在的精确页面？
- 一段时间后，类比是否提高了无提示回忆或问题迁移？

类比有效性应通过学习结果验证，不能仅凭“听起来形象”判断。

## 何时不使用

- 已有简短公式或示例比类比更直接；
- source 系统本身也不熟悉；
- 高风险决策会因类比省略边界而误导；
- 多个关键关系无法用同一 source 一致映射。

此时可改用最小例子、反例、流程图或对照表。
