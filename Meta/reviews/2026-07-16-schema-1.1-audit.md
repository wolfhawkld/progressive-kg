# 复核：Schema 1.1 全库审计

> 记录 Schema 1.1 迁移、事实校正、链接治理与 Obsidian Home 入口验收结果。

## 触发与范围

本次复核由“整体分析现有三层渐进披露知识库并按建议逐步修复”触发。范围覆盖 47 个内容页、19 个 MOC、Home 仪表盘、Schema、模板、lint、CSS 和 Obsidian 配置说明；`raw/` 作为不可变来源层未改写。

- 主入口要求：保留 `home.md` 的网页式 Reading View，图谱仅作辅助。
- 内容要求：修复结构漂移、错误链接、绝对化事实和空来源。
- 系统要求：默认只读校验，形成可追溯 review/log 闭环。

## 核验结果

审计先建立 Schema 1.1，再迁移全部页面，最后按来源逐页校正高风险事实。

## 主要决策

本次修改保持导航层和内容层相互独立，避免再用同一个 `level` 同时表达目录与披露深度。

- Home → Domain MOC → Subdomain MOC → Note 负责导航。
- summary/定义框 → L2 → L3 负责单页披露。
- `关系网络` 是人工关系的唯一维护位置。
- `updated` 与 `verified` 分离，格式迁移不冒充事实核验。
- Home 按 `updated` 展示最近修改，并按生命周期字段形成复核队列。
- 外部来源优先原始论文、规范和官方文档；个人假设保留低置信度。

## Home 入口验收

当前本机配置显示主 workspace leaf 打开 `home.md`，mode 为 `preview`；Dataview 已安装，`hierarchy-visual` 已启用，源/活跃 CSS 内容一致。

- Home 域导航和 About 链接均由 lint 验证为存在。
- 最近更新查询使用 `updated`，不再把核验日期当修改日期。
- 待审阅查询覆盖 maturity、confidence、verified、sources 和 review_due。
- 未启动 Obsidian GUI 做像素级截图比对；换机器后的人工验收步骤写入 setup guide。

## 未解决风险

结构校验不能替代所有运行时和外部证据验证，以下项目保留为下一轮触发条件。

- lint 只验证外部 URL 形式，不主动联网逐个检查可访问性或内容漂移。
- Dataview/Obsidian 插件升级后需按 `review_due` 重新做 Home 动态渲染验收。
- AGI 路径和类比导航法仍是低证据的研究/方法假设，需要实验而非提高文字置信度。
- 快速变化的工具页设置较短复核周期；数学基础按年复核。
